================================================================================
  SJS — КАТАЛОГ ПОЛЬЗОВАТЕЛЬСКОГО ФУНКЦИОНАЛА
  Единый источник правды: STUDENT + EMPLOYER + ADMIN + Public
  Базовый URL: http://localhost:3000/api
  Детали API по ролям: STUDENT_*, EMPLOYER_*, ADMIN_* (ссылаются на §F-XX)
  Интеграции: docs/INTEGRATIONS.md
  Версия: 1.0  |  Дата: июнь 2026
================================================================================

Легенда статусов (после аудита):
  OK            — REST-флоу работает с Postgres + Redis (+ S3 где нужно)
  Degraded      — работает без внешнего ключа (demo / skip / 503 на AI)
  Data-dependent — нужен seed или admin-данные
  Limitation    — известное ограничение, не баг

Prod checklist (webhooks):
  KASPI_WEBHOOK_SECRET, TELEGRAM_WEBHOOK_SECRET — обязательны в production

§F-01  Аутентификация (JWT)
  Роли:     STUDENT, EMPLOYER, ADMIN
  API:      POST /auth/register, /auth/login, /auth/refresh, /auth/logout
  Env:      JWT_SECRET (≥32), JWT_EXPIRES_IN, REFRESH_TOKEN_DAYS
  Статус:   OK
  Проверка: test/auth.e2e-spec.ts, test/api-smoke.e2e-spec.ts
  Примечание: ADMIN не регистрируется через API — npm run seed:bootstrap

§F-02  Профиль аккаунта
  Роли:     STUDENT, EMPLOYER, ADMIN (минимальный)
  API:      GET /users/me, PATCH /users/me
  Env:      S3_* для avatar/logo presign URL; OPENAI для embedding профиля (фон)
  Статус:   OK
  Проверка: test/api-smoke.e2e-spec.ts (student me)
  EMPLOYER: logoStorageKey, logoUrl, telegramChatId (маска ***linked***)
  STUDENT:  avatarStorageKey, avatarUrl, firstName, university, …

§F-03  Справочники (каталог)
  Роли:     Public (активные); ADMIN — CRUD через /admin/*
  API:      GET /cities, /job-categories, /tags, /catalog/job-form
  Ответ:    imageStorageKey + imageUrl (presigned GET) для городов и категорий
  Env:      npm run seed:catalog
  Статус:   Data-dependent без seed
  Проверка: test/api-smoke.e2e-spec.ts GET /catalog/job-form

§F-04  Загрузка файлов и медиа URL
  Роли:     STUDENT, EMPLOYER (JWT upload); Public (GET media)
  API:      POST /upload/presign, GET /media/url?storageKey= (JWT, own key only)
  Env:      S3_ENDPOINT, S3_BUCKET, S3_ACCESS_KEY, S3_SECRET_KEY
  Статус:   OK с MinIO; без S3 — upload fails
  Проверка: test/features.e2e-spec.ts (media url validation)

§F-05  Уведомления
  Роли:     STUDENT, EMPLOYER
  API:      GET /notifications, GET /notifications/unread-count,
            PATCH /notifications/:id/read
  Каналы:   Telegram (TELEGRAM_BOT_TOKEN), SMTP, Twilio SMS — side channels
  Статус:   OK in-app; каналы Degraded без ключей
  Проверка: создаётся при смене статуса отклика (employer-admin e2e)

§F-06  Привязка Telegram
  Роли:     STUDENT, EMPLOYER
  API:      POST /telegram/link-token; webhook POST /telegram/webhook
  Env:      TELEGRAM_BOT_TOKEN, TELEGRAM_BOT_USERNAME, TELEGRAM_WEBHOOK_SECRET
  Статус:   OK link-token; bot Degraded без token
  Проверка: test/employer-admin.e2e-spec.ts telegram link-token

§F-07  Геймификация
  Роли:     STUDENT (основной), EMPLOYER (read), Public leaderboard
  API:      GET /gamification/me, GET /gamification/leaderboard
  Env:      Postgres
  Статус:   OK
  Проверка: очки начисляются при отклике, тесте, отзыве (unit + e2e косвенно)

  Env:      npm run seed:jobs (демо-вакансии); OPENAI_API_KEY для embeddings/matchScore
  Статус:   OK listing; matchScore Degraded без OpenAI

§F-10  Отклики на вакансии
  Роли:     STUDENT (apply, list mine, detail, withdraw); EMPLOYER (list by job, detail, status)
  API:      POST /applications (jobId, resumeDraftId?, coverLetter?), GET /applications/me,
            GET /applications/:id, PATCH /applications/:id/withdraw (STUDENT),
            GET /applications/job/:jobId?status=, PATCH /applications/:id/status
  Env:      OPENAI для employerScore (фон, BullMQ)
  Статус:   OK
  Проверка: test/features.e2e-spec.ts, test/employer-admin.e2e-spec.ts
  Контракт: job, student, studentProfile, employerScore; hasReviewed (STUDENT only);
            resumeDraftId + resume summary (title, pdfUrl) в ответе;
            валидация requiresResume/requiresCoverLetter с вакансии

  Cron:     каждые 6ч при HH_IMPORT_ENABLED=true
