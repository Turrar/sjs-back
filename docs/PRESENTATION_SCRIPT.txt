================================================================================
  SJS — PRESENTATION SCRIPT
  Сценарий презентации для инвесторов / партнёров / университетов
================================================================================
  Формат: слайд → минимум текста на экране → что говорить вслух
  Время: ~8–10 минут
================================================================================

  "Добрый день. Сегодня я хочу рассказать вам о продукте,
  который решает проблему, с которой сталкивается каждый
  второй студент в Казахстане."


────────────────────────────────────────────────────────────────────────────────
  СЛАЙД 2 — ИСТОРИЯ. ЗНАКОМЬТЕСЬ: АЛИБЕК
────────────────────────────────────────────────────────────────────────────────

  НА ЭКРАНЕ:
  ┌──────────────────────────────────┐
  │  Алибек, 3 курс, Алматы          │
  │                                  │
  │  Хочет работать.                 │
  │  Не знает — где.                 │
  │  Не знает — когда.               │
  └──────────────────────────────────┘

  ГОВОРИТЬ:
  "Алибек — студент третьего курса. Ему 20 лет.
  Он учится в Алматы, хочет подрабатывать — покрыть аренду,
  купить ноутбук, получить первый опыт.

  Он заходит на HeadHunter. Видит сотни вакансий.
  Большинство — полный день. Остальные — 'график обсуждается',
  а на деле это значит 'приходи каждый день'.

  Алибек откликается на 15 вакансий за месяц.
  Проходит 4 собеседования.
  На каждом — один и тот же вопрос:
  'Когда вы можете выйти?'
  И каждый раз — тупик. Его расписание не совпадает."


────────────────────────────────────────────────────────────────────────────────
  СЛАЙД 3 — РЕАЛЬНАЯ ПРОБЛЕМА
────────────────────────────────────────────────────────────────────────────────

  НА ЭКРАНЕ:
  ┌──────────────────────────────────┐
  │  600 000 студентов в Казахстане  │
  │                                  │
  │  Каждый второй хочет работать    │
  │  Каждый третий — не может найти  │
  │                                  │
  │  Причина: расписание             │
  └──────────────────────────────────┘

  ГОВОРИТЬ:
  "Это не проблема одного Алибека.
  В Казахстане более 600 тысяч студентов.
  По данным социальных опросов — больше половины
  хотят совмещать учёбу и работу.

  Но ни одна платформа — ни HeadHunter, ни SuperJob,
  ни LinkedIn — не знает расписание студента.
  Они показывают вакансии всем подряд.
  Фильтра 'я свободен во вторник и пятницу после 14:00' — нет нигде.

  Работодатель тоже страдает: получает десятки откликов,
  тратит часы на переписку — и узнаёт на собеседовании,
  что студент занят именно тогда, когда нужен."


────────────────────────────────────────────────────────────────────────────────
  СЛАЙД 4 — РЕШЕНИЕ
────────────────────────────────────────────────────────────────────────────────

  НА ЭКРАНЕ:
  ┌──────────────────────────────────┐
  │  SJS знает расписание.           │
  │                                  │
  │  И подбирает работу под него.    │
  └──────────────────────────────────┘

  ГОВОРИТЬ:
  "Вернёмся к Алибеку. Он регистрируется в SJS.
  Загружает PDF расписания из университетского портала —
  один файл, одно действие.

  Система автоматически разбирает расписание через AI,
  определяет все занятые слоты и убирает из ленты
  вакансии, которые пересекаются с парами.

  Теперь Алибек видит только то, куда он реально может выйти.
  Не 500 вакансий. А 40 — но каждая подходит."


────────────────────────────────────────────────────────────────────────────────
  СЛАЙД 5 — КАК ЭТО РАБОТАЕТ ДЛЯ АЛИБЕКА
────────────────────────────────────────────────────────────────────────────────

  НА ЭКРАНЕ:
  ┌──────────────────────────────────┐
  │  1. Загрузил расписание          │
  │  2. Получил подходящие вакансии  │
  │  3. AI написал письмо            │
  │  4. Прошёл тест по JavaScript    │
  │  5. Работодатель написал первым  │
  └──────────────────────────────────┘

  ГОВОРИТЬ:
  "Алибек нашёл вакансию ассистента разработчика.
  Система показала ему, что он совпадает с требованиями на 78%.

  Он нажал 'Откликнуться'. Система предложила —
  хотите, GPT напишет сопроводительное письмо?
  Алибек выбрал тон, язык, нажал кнопку.
  Письмо готово за 10 секунд.

  Перед откликом он прошёл тест по JavaScript прямо в приложении.
  Набрал 84%. Бейдж появился в профиле.

  Работодатель открыл профиль Алибека, увидел бейдж,
  AI-скоринг и GitHub с реальными проектами.
  И написал первым — через 20 минут."


────────────────────────────────────────────────────────────────────────────────
  СЛАЙД 6 — ЧТО ПОЛУЧАЕТ РАБОТОДАТЕЛЬ
────────────────────────────────────────────────────────────────────────────────

  НА ЭКРАНЕ:
  ┌──────────────────────────────────┐
  │  Только доступные кандидаты      │
  │  AI-оценка каждого отклика       │
  │  Верифицированные навыки         │
  │  Чат + видеоинтервью             │
  │  Трекер стажировки               │
  └──────────────────────────────────┘

  ГОВОРИТЬ:
  "Работодатель в SJS — это не просто 'разместить вакансию'.

  Каждый отклик приходит с оценкой 0 до 100 —
  насколько студент подходит по навыкам и опыту.
  Не нужно читать каждое резюме.

  Видит, когда студент свободен — без лишних вопросов.
  Запускает видеоинтервью прямо в приложении — без Zoom.
  Нанял — открывает трекер стажировки: ставит задачи,
  видит журнал часов, в конце оставляет официальный отзыв.

  Весь цикл: отклик → найм → стажировка → отзыв —
  в одном месте."

  ГОВОРИТЬ:

  Ни один из них за 20+ лет не сделал

  Потому что им не нужно — они работают
  с профессионалами, которые управляют своим временем сами.

  Студент — другой пользователь. Его время структурировано
  университетом. И это наша уникальная точка входа.

  Мы не конкурируем с HeadHunter за всю аудиторию.
  Мы берём ту нишу, которую они никогда не займут —
  потому что она требует другой логики продукта."

────────────────────────────────────────────────────────────────────────────────
  СЛАЙД 9 — ТЕХНОЛОГИИ (КОРОТКО)
────────────────────────────────────────────────────────────────────────────────

  НА ЭКРАНЕ:
  ┌──────────────────────────────────┐
  │  NestJS  ·  TypeScript           │
  │  PostgreSQL + pgvector           │
  │  OpenAI  ·  BullMQ  ·  Redis     │
  │  MinIO (S3)  ·  Socket.IO        │
  │  Docker  ·  Swagger              │
  └──────────────────────────────────┘

  ГОВОРИТЬ:
  "Коротко о технической стороне.

  Бэкенд на NestJS — это модульный, масштабируемый TypeScript.
  База данных — PostgreSQL с расширением pgvector,
  которое позволяет хранить и искать AI-векторы прямо в SQL.

  Фоновые задачи — BullMQ с Redis: эмбеддинги, скоринг,
  парсинг расписания работают асинхронно, не тормозят API.

  Файлы — MinIO, S3-совместимое хранилище, работает локально
  и в облаке без изменений в коде.

  Чат в реальном времени — Socket.IO.
  API задокументирован через Swagger — доступен по /api/docs.

  Всё поднимается локально командой docker compose up.
  Никаких внешних зависимостей для старта."

  СОВЕТЫ ДОКЛАДЧИКУ
================================================================================

  - Слайды — это фон. Смотреть на аудиторию, не на экран.
  - Паузы после ключевых фраз — они работают лучше слов.
  - История Алибека — говорите как про реального человека.
  - Слайд 7 (амбиции) — самое важное место. Не торопитесь.
  - Слайд 9 (стек) — 30 секунд максимум, детали по запросу.
  - Финал — одна пауза перед "600 000 ждут". Дайте осесть.
