# Совместимость расписания студента и графика вакансии

Реализация: [`ScheduleCompatibilityService`](../src/modules/schedule/schedule-compatibility.service.ts).  
Хранение: занятость студента — строки `ScheduleSlot` (занятые интервалы); вакансия — `Job.workWindows` (JSON массив `JobWorkWindowJson`).

## Представление времени

- **День недели:** `0` = понедельник … `6` = воскресенье (как в коде сущностей).
- **Время:** минуты от полуночи, полуинтервал `[startMinute, endMinute)` в пределах `[0, 1440)`.

## Модель занятости студента

- В БД хранятся **занятые** интервалы (пары). Свободное время на день = `[0, 1440)` минус объединение пересекающихся занятых интервалов (merge).

## Модель работодателя

- `work_windows` — список окон, когда требуется присутствие сотрудника (например вечерние смены вт–чт 18:00–22:00). Все окна из списка должны быть выполнимы одновременно для одной недели.

## Правило совместимости (MVP)

Для каждого окна вакансии `W` на день `d` существует **один** свободный интервал студента `F` на том же `d`, такой что `W ⊆ F` (границы окна работы целиком лежат внутри одного свободного блока).

- Если у вакансии нет `work_windows` (null/пусто) — считается совместимой с любым расписанием.
- Частичные пересечения без полного покрытия окна считаются **несовместимыми** в MVP (можно ослабить позже: минимальное пересечение по минутам).

## Пример

- Студент занят пн 09:00–12:00 и 14:00–18:00 → свободно пн 12:00–14:00 и 18:00–24:00 (как объединённые интервалы).
- Вакансия требует пн 13:00–14:00 — **совместимо** (окно внутри 12:00–14:00).
- Вакансия требует пн 13:00–15:00 — **несовместимо** (15:00 не покрыто одним свободным блоком).

## Интеграция в поиск вакансий

`JobsModule` при фильтре `compatibleWithSchedule=true` загружает слоты студента, вызывает `ScheduleCompatibilityService.isJobCompatible(busySlots, job.workWindows)` и отбрасывает ложные.
